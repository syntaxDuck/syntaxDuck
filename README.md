# Kameron Comer · syntaxDuck

Software engineer drawn to the low-level stuff — the parts most people abstract away.

I write C when I want to understand something, Python when I want to build something fast, and TypeScript when the frontend demands it. ### How I think about code

I'd rather understand one layer deeper than use a convenient abstraction I can't explain. That's why I build text editors instead of just using them, and raycasters instead of just playing the games. Clean code, to me, means a future reader can follow the logic without a tour guide.
